# Knowledge Creator 版本迭代计划

## 迭代策略 (参考开源最佳实践)

### 1. 语义化版本控制 (SemVer)
- **MAJOR**: 破坏性变更 (v1.0.0 → v2.0.0)
- **MINOR**: 新功能 (v1.0.0 → v1.1.0)
- **PATCH**: 修复 (v1.0.0 → v1.0.1)

### 2. 迭代维度
每个版本只关注一个维度：
- 🔧 代码质量 (重构、优化)
- ✨ 功能增强 (新特性)
- 🐛 缺陷修复 (Bug修复)
- 📚 文档完善
- ⚡ 性能优化
- 🧪 测试覆盖
- 🔒 安全加固
- ♿ 可访问性

### 3. 迭代节奏
- PATCH: 每天 1-3 个
- MINOR: 每周 1-2 个
- MAJOR: 每月 1 个
